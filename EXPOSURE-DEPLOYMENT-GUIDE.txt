唐人日报“我要曝光”部署说明 v29.12

功能：
1. 当事人公开投稿文字。
2. 支持最多12个图片/视频文件。
3. 图片单张限制10MB，视频单个限制200MB。
4. 投稿发布后立即公开。
5. 每篇曝光内容支持网友评论。
6. 后台可把内容标记为：已发布、争议中、已解决、隐藏、撤稿。
7. 后台可显示、隐藏或删除评论。
8. 投稿人的联系方式只在后台显示，不会通过公开接口返回。

部署步骤：
A. Supabase
1. 打开 Supabase 项目。
2. 进入 SQL Editor。
3. 打开本包中的 supabase-exposure-setup.sql，复制全部内容并运行一次。
4. 运行成功后，Table Editor 中会出现：
   exposure_posts
   exposure_media
   exposure_comments
5. Storage 中会出现 exposure-media 公共存储桶。

B. Netlify 环境变量
Netlify -> Project configuration -> Environment variables，确认已有：
SUPABASE_URL=https://fwiznbpsqkfgkvyznebz.supabase.co
SUPABASE_ANON_KEY=你的 Supabase publishable/anon key
SUPABASE_SERVICE_ROLE_KEY=你的 Supabase service_role key

SUPABASE_SERVICE_ROLE_KEY 只能放在 Netlify 环境变量中，不能写进前端文件或公开。
当前项目已有 generate-cover 函数使用该变量；如果 AI 封面生成目前可用，通常已经设置。

C. GitHub/Netlify
1. 将本包完整上传到 GitHub main 分支，覆盖同名文件。
2. 提交说明：Add public exposure posts media and comments v29.12
3. 等 Netlify Published。
4. 打开 /exposure.html 测试投稿。
5. 打开 /admin/exposure.html 测试后台管理。

D. 测试
1. 发布一条文字曝光。
2. 发布一条带图片的曝光。
3. 发布一条短视频曝光。
4. 打开曝光详情，发表评论。
5. 登录 /admin/exposure.html，将内容改为“争议中”，刷新前台确认标签变化。
6. 将一条评论隐藏，刷新详情确认评论消失。

重要：
- 视频直接从浏览器上传至 Supabase Storage，不经过 Netlify Functions，因此支持较大文件。
- 公开投稿会立即上线。
- 联系方式保存在 exposure_posts.author_contact，只有管理员可在后台读取。
